第〇章

为什么现在做（Why Now）

战略时机判断

这是一个时机窗口判断，而非功能迭代。以下四点说明为什么这个功能需要在当前阶段落地，而不是再等一个版本。

理由一
流量飞轮已从增长期切换到留存深耕期
红果短剧的用户规模已进入平台级体量。在这个阶段，「拉新」的边际成本快速上升，「留存」和「时长」成为核心竞争指标。当前大结局后的用户自然流失是一个已知的流失漏斗，且缺乏任何专属的情绪承接产品设计——这是一个明显的产品空白，也是留存指标提升的最直接抓手。
理由二
End Screen 广告库存空白至今未被开发
完播大结局后的高意愿用户，是广告转化率最高的流量池之一。用户在情绪最高涨的时刻主动决策（看广告解锁彩蛋），转化质量显著高于信息流中被动曝光的广告。这块高净值广告库存目前是空置的，是直接可盘活的商业化增量。
理由三
竞争窗口期有限
微短剧赛道的头部平台正在加速布局大结局后的用户运营产品。先行落地的平台将在「彩蛋供给链」（与承制方的合作协议）和用户心智上建立壁垒。这个功能的价值不仅在于上线后的直接收益，还在于抢占生态卡位。
理由四
基础设施已就绪
红果当前的激励广告 SDK、创作者后台以及内容审核中台，均已具备支撑本功能落地所需的基础能力，不需要从零建设核心组件，研发周期可控。
第一章

需求背景

1.1 核心用户痛点

短剧用户在完播大结局后，面临两种典型情绪状态，当前产品设计对二者均无差异对待，且缺乏任何专属的情绪收口机制：

😭
满足型（HE / 爽剧结局）
剧情完美收尾，情绪处于高峰，强烈希望延续与角色的情感联结。但平台缺乏承接手段，用户直接退出 App 或跳转抖音、小红书搜演员，形成不必要的流失。
😤
意难平型（BE / 开放结局 / 烂尾剧）
情绪是愤怒和失落。若此时强推花絮，反而加重负面体验。MVP 阶段对这类用户统一触发，通过 A/B 数据观察不同情绪类型剧集的转化差异，为第二期情绪分层策略提供依据。
1.2 商业机会
  • 完播大结局用户是广告转化意愿最高的流量池，当前 End Screen 广告库存未被有效开发。
  • 短剧 IP 的「长尾内容价值」（花絮、番外、品牌定制小剧场）尚未被系统化产品化，存在明显的变现空白。
  • 承制方侧存在大量低成本的边角料物料（NG、花絮），缺乏变现渠道，供给侧有充分的合作动机。
第二章

需求目标与成功指标

2.1 核心目标
  • 商业化增收（P0）：开拓彩蛋解锁场景下的高净值激励广告库存，提升该场景 eCPM。
  • 用户留存（P0）：缩短大结局后的流失漏斗，提升完播后次日留存率。
  • IP 生态沉淀（P1）：将「阅后即焚」的短剧流量引流至剧集详情页，延长单 IP 的长尾浏览生命周期。
  • 生态共建（P1）：通过分账机制激励承制方主动供给高质量花絮物料，形成内容供给飞轮。
2.2 成功指标（A/B 实验验收标准）

以下指标为实验组 vs 对照组的对比目标，具体基线值需与 Data 团队对齐后填写。

2.3 Kill 条件（A/B 提前终止规则）

以下任一条件触发时，立即暂停实验并回滚：

🛑

🛑

🛑

第三章

MVP 范围与分期规划

第一期 MVP（当前需求范围）
  • 剧集白名单：仅对 CMS 后台配置了 has_easter_egg: true 的 S 级独播/自制剧及商业化定制剧开放，A 级及以下不纳入。
  • 解锁机制：唯一路径为「观看 30 秒激励视频广告」（Ads-Only），无金币解锁，聚焦商业化转化，降低法务与经济系统复杂度。
  • 物料双轨制：视频轨（承制方上传花絮）为主，NLP 词云战报为兜底，确保白名单剧集均有内容可解锁。
  • 情绪分层：MVP 阶段统一触发，通过 A/B 数据观察不同情绪类型剧集的转化率差异。
  • 彩蛋后续播：彩蛋视频播放完毕后，无缝自动续播算法推荐的下一部同类剧集第一集。
第二期迭代方向（MVP 验证通过后）
  • 情绪分层触发策略：基于剧集标签（爽剧/虐剧/烂尾）配置差异化气泡文案和触发策略。
  • 算法特征接入：引入 IPW 因果推断校正后的解锁特征，进入推荐模型训练。
  • 创作者质量分体系：基于用户评分数据建立承制方彩蛋质量评分，与合作优先级挂钩。
  • 广告疲劳度检测：建立 7 日周期疲劳检测机制，对高频解锁用户降级为词云战报。

4.1 播放器内轻量引流——限时气泡（非阻塞式）
触发时机 / UI 表现 / 特殊模式降级
4.1.1 触发规则
  • 触发时机：大结局单集播放进度到达 90% 时触发气泡下发。
  • 倍速适配：根据用户当前播放倍速动态调整触发进度点——1.0× 在 90% 触发；1.5× 在 85% 触发；2.0× 在 80% 触发。确保用户在情绪峰值期（而非剧情已过去后）看到提示。
  • 并发冲突预防：当播放进度到达 98% 时，客户端进入「准备连播」锁定状态，气泡的点击事件降级处理——不再唤起半屏详情页，只展示 Toast 提示「彩蛋已保存，稍后在详情页查看」，并在该剧详情页花絮 Tab 增加红点标记。此设计通过提前消除冲突时间窗口，避免复杂的并发状态仲裁。

4.3.2 解锁机制（Ads-Only）
  • 列表展示：单列或双列瀑布流，每条花絮右上角带 🔒 图标，标注「未解锁」状态，展示时长和内容标题。
  • 解锁操作：点击带 🔒 的条目 → 底部弹出确认面板 → 展示「观看 30 秒视频，免费解锁此花絮」按钮（防连点，点击后 1.5 秒内禁止重复触发）。
  • 广告完播 → SDK 下发回调 → 解锁状态写入账号资产库 → 🔒 解除 → 花絮视频直接播放。
4.3.3 账号级解锁状态记忆
  • 解锁记录绑定账号 ID，跨设备、跨端永久生效，不重复解锁。
  • 扣费前向服务端校验当前解锁状态（不依赖本地缓存），防止多端同时解锁的重复扣费场景。
存储架构提示（PM 提示研发重点）：解锁状态查询属于「高频读、低频写、99% 查询结果为空」的典型场景，请研发团队评估 Bloom Filter + Redis 缓存方案，防止缓存穿透导致数据库压力。
▶
4.4 彩蛋播放完毕后——无缝续播下一部剧
情绪收口完成后，算法接管，保障大盘连播率
  • 彩蛋视频播放完毕后，页面无缝自动全屏化，直接衔接算法推荐的下一部同类剧集的第一集，无需用户额外操作。
  • 续播逻辑与常规大结局后的推荐连播链路复用同一套算法策略，保持一致的用户连播心智。
  • 若用户在彩蛋播放期间主动返回详情页（如快进/回看操作），半屏页面在彩蛋结束后保留 3 秒，展示「下一部剧」强引导按钮后自动关闭，恢复推荐流。
设计意图：彩蛋作为情绪缓冲垫，在满足用户完播后对剧集 IP 的情感延续需求的同时，将用户顺滑地引回主干连播链路。彩蛋结束 ≠ 用户旅程结束，而是交接给算法的新起点。
第五章

物料质量闭环机制

上传前 · 分发前 · 解锁后

物料质量是整个功能的体验底线。一次低质量解锁（用户看完广告，结果是手机摇晃废片）足以摧毁用户对该功能的信任。需要在三个节点分别设卡。

5.1 上传前：B 端准入硬标准（系统自动拦截）
检测维度最低标准不达标处理
视频时长≥ 20 秒，≤ 3 分钟系统拒绝上传，提示原因
画面分辨率≥ 720P系统拒绝上传
画面重复度（与正片比对）重复率 < 30%系统拒绝，标注重复时间段
音频完整性须有人声或配乐，纯静音拒绝系统拒绝上传
内容分级标注全年龄 / 限制级，必选缺失则无法提交
画面重复度检测技术说明：每 5 秒抽取一帧，与正片帧库做感知哈希（pHash）相似度比对，超过阈值则标记为重复内容，告知创作者具体重复时间段。可复用抖音系图像识别中台能力，无需从零建设。
5.2 分发前：审核 SLA 分级保障
  • S 级剧集（合同约定）：拍摄结束后 72 小时内上传物料，平台开放绿色审核通道，工作日内 4 小时完成，节假日提前排班。此条款写入制作合同，作为触发分账权益的前提条件。
  • A+ 级剧集：走常规 24 小时审核流程。
  • 所有彩蛋内容接入与正片共享的「机审 + 人审」双重合规流程，无通过标识的物料静默不予分发。品牌定制小剧场内容，全程强制悬浮「广告」标识，不可关闭。
5.3 解锁后：用户评分反馈回路
  • 花絮播放完毕后弹出轻量评分组件（不强制，3 秒后自动消失）：「👍 值得解锁」/ 「👎 有点失望」。
  • 差评率 > 30%：触发运营复审通知，该物料暂停分发。
  • 差评率 > 60%：系统自动下架，扣除对应期间分账，通知创作者重新上传。
  • 评分数据沉淀为「承制方彩蛋质量分」，第二期纳入合作优先级评估体系。
第六章

用户级频控机制

重度追剧用户可能每天完播多部剧的大结局，缺乏频控保护会导致广告疲劳加速，最终使高价值用户主动屏蔽该功能。

LAYER 01
单剧级
账号级永久生效

同一用户对同一部剧的彩蛋，解锁后永久记忆，不再触发解锁广告流程。

已覆盖于账号级状态记忆（见 4.3.3），Bloom Filter 过滤 + Redis 缓存，支持跨设备查询。
LAYER 02
当日频控
自然日 0 点重置

同一自然日内，激励广告解锁上限 2 次（建议起始值，通过 A/B 实验验证最优值）。

达到上限后：气泡继续触发，但进入详情页后，解锁面板改为展示词云战报（静态图片，一键免费解锁），不再加载激励广告。
LAYER 03 · 第二期
周期疲劳检测
7 日滚动窗口

连续 7 天触发日频控上限的用户，标记为「广告高频用户」，降低广告曝光频次。

触发后：气泡改为情感导向文案，优先推送词云战报，以保护长期留存换取短期广告收益。
6.2 广告库存穿底兜底（Fill Rate = 0）

若用户点击解锁后，广告联盟接口超时（1.5 秒）或返回无广告缓存：

  • 前端 Toast 提示：「系统开了个小差，本次特权免费为您解锁」，直接下发免广告观看权益，解锁状态正常写入。
  • 该次降级触发计入监控，单日穿底率超过阈值时触发运营告警，确保广告库存预留充足。
第七章

工程稳定性与异常兜底

7.1 播放器单实例架构（防 OOM）

当用户点击气泡拉起半屏详情页时，客户端执行以下操作：

  • 立即暂停并销毁底层推荐流正片播放器实例，释放内存。
  • 底层画面用该集最后一帧的高斯模糊封面图接管。
  • 详情页内的花絮视频复用全局唯一播放器实例。
确保无论在何种设备上，同一时刻内存中只存在一个播放器实例，彻底消除低端机因双播放器并存导致的 OOM 崩溃风险。
7.2 CDN 预热与峰值熔断

爆款短剧大结局上线时，可能引发百万级用户在短时间内同时触发彩蛋请求。CMS 配置物料时增加「计划上线时间」字段，触发以下自动化流程：

时间节点系统动作
T−48hCDN 预热指令下发，彩蛋视频推送至该剧热度 TOP 城市边缘节点
T−24h广告投放系统收到通知，预估并锁定对应量级广告库存，避免大结局时段广告穿底
T−0监控看板自动开启，彩蛋视频起播失败率 > 3% 触发 On-Call 告警

视频起播失败时，降级为 NLP 自动生成的情绪词云战报（静态图），CDN 压力极低，保障用户有内容可解锁。

全站异常
7.3 广告 SDK 部分完播处理规范
中断场景判定规则处理方式
网络中断广告 SDK 请求失败 / 缓冲超时需重新拉起广告，从头完整观看 30 秒
App 压后台（非最后 5 秒）广告暂停，时长不累积回到 App 后从中断处继续播放
App 压后台（最后 5 秒内）视为有效完播回到 App 后直接下发解锁权益
电话来电（任意时间段）时长不累积，广告暂停挂断后从中断处继续播放
广告回调网络中断兜底：用户广告完播后，若网络断开导致 SDK 回调未发出，客户端需在本地缓存「完播时间戳 + 广告 ID」，联网恢复后自动补发解锁请求。服务端对同一用户同一广告 ID 做幂等校验，防止重复解锁。
7.4 离线模式处理
离线状态下的降级策略
  • 客户端检测到无网络状态（或广告 SDK 超时 2 秒）时，不触发气泡，不展示评论区置顶卡片，避免拉起无法加载的交互链路。
  • 下沉市场（三线及以下城市）网络不稳定场景高发，此降级策略覆盖红果核心用户群体。
「延迟触达」补通知机制

用户在离线状态下完播大结局时，客户端本地记录：drama_id + finale_watched_offline: true + timestamp。

下次用户联网打开 App 时（T+1 次启动），触发一条 App 内消息中心气泡通知（不发 Push，避免打扰）：

「你上次离线看完了《XXX》，独家花絮还在等你哦 🎁」→ 点击直跳该剧详情页花絮 Tab。

此机制确保下沉用户完播大结局的行为价值不因网络条件而丢失，提升彩蛋功能在全体量用户中的实际覆盖率。

第八章

算法策略与推荐协同

8.1 第一期：规则驱动触发（不上预测模型）

MVP 阶段大结局场景的用户行为样本稀疏，不具备训练精准预测模型的条件。采用强规则触发保证样本质量：

  • 触达条件：该剧观看完成度 100% 且发生过至少 1 次点赞或评论互动的用户。
  • 排除条件：当日已触发 2 次广告解锁的用户（见频控机制），触达时展示词云战报。
8.2 完整行为数据记录（为第二期因果推断打基础）

MVP 阶段的核心任务之一是为第二期的特征工程打好数据基础。需要完整记录两组行为：

  • 实验组曝光 + 点击数据：看到气泡且点击进入的用户（高意愿自选择群体）。
  • 实验组曝光 + 未点击数据：看到气泡但未点击的用户（同样高价值的对照参照）。
两组数据的完整留存，是第二期使用 IPW（Inverse Propensity Weighting，逆倾向加权）方法剥离自选择效应偏差的前提。直接用未经校正的点击用户特征训练模型，会系统性高估某类剧集吸引力，导致推荐收窄，与多样性 OKR 冲突。
8.3 弱特征输入策略（第一期安全边界）

MVP 阶段，彩蛋解锁行为不直接作为推荐模型的独立训练特征，而是作为用户现有兴趣标签体系的「置信度增强信号」：

  • 用户解锁了「霸总甜宠」类彩蛋 → 对应兴趣标签置信度 +1（不新建标签，不改变权重体系）。
  • 影响有限，不对推荐大盘造成系统性偏移，可安全观测 7 天数据再做第二期决策。
8.4 推荐多样性保护

在 A/B 实验期间，算法团队需同步监控推荐结果的内容多样性指标（ILD, Intra-List Diversity）：

  • 若实验组 ILD 显著低于对照组（低于对照组均值 5% 以上），立即暂停弱特征输入，等待 IPW 校正方案就绪。
  • 彩蛋播放完毕后的续播，通过推荐算法排序驱动，而非模型特征直接驱动，降低偏差引入风险。
第九章

合作方利益分配机制

9.1 物料供给激励
  • 在红果创作者后台开放「大结局彩蛋物料」独立上传入口，SOP 规范见第五章。
  • MVP 期成本兜底：针对参与首期试点的 S 级剧集承制方，平台一次性支付「物料采购费」（具体金额由商务团队与合作方协商），保障其不因参与试点产生额外成本亏损。
9.2 增量分账机制
  • 设立专项分账池：用户通过观看激励广告解锁特定剧集彩蛋所产生的直接广告收益，平台与该剧集版权方按约定比例结算分润（建议起始比例 7:3，具体需商务与法务确认）。
  • 分账范围仅限本功能产生的「增量广告收益」，不影响现有正片分账协议。
  • 分账数据在创作者后台透明展示，按自然月结算。
9.3 长效流量反哺
  • 彩蛋视频播放结束页面，下方展示「关注主创」及「查看该团队在红果平台的更多作品」按钮。
  • 流量反哺仅导向承制方在红果平台上架的作品，不为站外平台内容导流。此限制需在合作协议中明确约定。

  • 实验组：展示气泡 + 评论区置顶 + 花絮 Tab，激励广告解锁，彩蛋结束后无缝续播。
