Chapter 01

需求背景

1.1 问题所在

短剧用户在追完大结局后，常因剧情高潮骤停产生强烈的情绪落差——对演员、剧情幕后、同类内容的渴望在完播后五分钟内最为旺盛。而当前红果短剧在这一时刻没有任何专属设计承接这份情绪，用户大多切出 App 去微博搜演员或在抖音找同类剧，大结局完播是目前平台最大的未被承接的高价值流量节点。

1.2 为什么现在做（Why Now）
📦
供给侧时机成熟
承制方在短剧拍摄过程中产生大量花絮边角料，当前缺乏任何变现渠道。建立分账机制的窗口期已至，平台可以极低成本撬动优质物料主动供给，形成内容供给飞轮。
📊
用户行为数据验证了需求
大结局完播后，站外对演员名字和同类剧的搜索行为高度可观测，说明用户有强烈的情绪延伸需求。平台目前没有任何机制承接这部分流量，导致高价值用户持续流失至站外。
🎯
竞品窗口期有限
主流视频媒体的花絮体系以 PC/大屏为主，移动短剧场景普遍缺位；海外同类平台虽有幕后内容但依赖站外分发，商业化价值几乎为零。短剧赛道「大结局彩蛋」尚无成熟产品范式，先行者有机会率先建立与承制方的内容合作壁垒。
💰
广告库存结构优化
End Screen 场景目前广告库存利用率偏低。激励广告在用户主动触发场景下 eCPM 显著高于被动曝光，彩蛋解锁是天然的主动触发场景，具备开拓高净值广告库存的条件。
1.3 竞品分析
维度爱奇艺Reel Short红果彩蛋匣（本方案）
花絮内容形态「彩蛋剧情」「删减片段」独立上架，有成熟内容体系ReelTalk Backstage 系列，内容挂在 Spotify 播客和官网独立页花絮视频（承制方上传）+ 多类型（NG/番外/品牌定制小剧场）
大结局触达机制❌ 无主动触达，用户需自行寻找❌ 完全脱离主观看路径，无站内闭环✅ 90% 气泡 + 评论区置顶 + 底部半屏彩蛋 Tab
商业化闭环VIP 订阅制，无独立变现❌ 免费可看，内容价值完全放弃✅ 激励广告解锁（Ads-Only），独立开拓 End Screen 广告库存
移动端适配入口深，发现成本高移动端 App 内无花絮，只有站外跳转✅ 原生气泡 + 底部半屏 Tab，全程不离开 App
核心结论：爱奇艺和 Reel Short 共同证明了短剧用户对幕后花絮有真实消费意愿，但两者都错过了大结局完播时情绪最高的触达时机。红果彩蛋匣的差异化不在内容本身，而在于把已验证的内容形态嵌入一套精准的情绪峰值触达 + 广告变现机制。
Chapter 02

目标与成功指标

2.1 核心目标
  • P0 商业化增收：在大结局完播场景开拓激励广告库存，提升该场景 eCPM，日均新增 Impression 正向可观测。
  • P0 留存提升：缩短大结局后流失漏斗，大结局后次日留存率实验组不低于对照组。
  • P1 IP 长尾沉淀：将完播流量引流至剧集底部半屏彩蛋 Tab，延长单 IP 有效访问周期。
  • P1 内容生态共建：通过分账机制激励承制方主动供给花絮物料，建立内容供给飞轮。
2.2 核心指标（A/B 实验验收）

🛑
大结局后 1h 连播率实验组比对照组下降超 3%——气泡严重干扰主干连播链路。
🛑
App Crash 率实验组比对照组上升超 0.5%——工程稳定性红线。
🛑
彩蛋差评率（👎）连续 3 天超 40%——物料质量不合格，功能信任度崩溃。

4.2 三条用户故事
USER STORY 01 · 情绪留存
作为一个追完 108 集大结局的重度用户，我希望完播的那一刻能有个出口延续和剧集的情感联结——而不是只能退出 App 去微博搜演员，因为我在这部剧上投入了将近一个月。
USER STORY 02 · 内容深探
作为一个对剧情和角色有强烈好奇心的追剧用户，我希望能看到正片之外的花絮和番外——NG、台前幕后、主创故事，因为大结局让我意犹未尽，这种「多看一点」的欲望在完播后五分钟内最强烈。
USER STORY 03 · 离线无缝
作为一个在地铁或信号差的场所追剧的用户，我希望即使离线状态下看完大结局，下次打开 App 时也能收到提醒——那种「有东西等着我」的感觉本身就是回来的理由。

  • S2S Callback 为准：广告完播校验以服务端收到广告商回调为准，客户端本地仅辅助记录。用户杀后台重开需重看广告，防黑灰产免广告刷解锁漏洞。

关键流程

6.1 Happy Path（6 步）
1
触发气泡
用户观看大结局，进度达 90%。客户端向后端请求「是否触发」，后端校验白名单 + 频控 + 解锁历史，满足条件后在左下角「选集」按钮正上方下发气泡，带呼吸灯动效。
2
拉起彩蛋 Tab
用户点击气泡，底部半屏打开并自动切换至「彩蛋」Tab。底层播放器暂停，实例销毁，封面图接管，防 OOM。若从评论区点击：先 Dismiss 评论区，再拉起半屏。
3
选择彩蛋发起解锁
用户点击带 🔒 的彩蛋条目，底部弹出确认面板（防连点，1.5s 内禁重复触发）。请求广告 SDK，1.5s 超时则穿底免费解锁兜底。
4
广告完播
用户完整观看 30 秒激励广告。以服务端 S2S Callback 为准；用户杀后台重开需重看；网络中断本地缓存时间戳，联网后补发，服务端幂等校验。
5
解锁成功 → 彩蛋播放
SDK 回调 → 解锁状态写入账号库（Bloom Filter + Redis + MySQL）→ 🔒 解除 → 花絮视频自动播放。账号级永久记忆，跨设备生效，不重复解锁。
6
收口 → 无缝续播
彩蛋播完，展示续播推荐 + 演员关注 + 新剧预约三路收口，3 秒后自动起播算法推荐的下一部剧，回归主干连播链路。

Chapter 08

算法策略与推荐协同

8.1 第一期：强规则驱动，不上预测模型

MVP 阶段大结局场景行为样本稀疏，贸然训练预测模型会引入不可控偏差。采用强规则触发：该剧完成度 100% 且发生过至少 1 次点赞或评论互动的用户，排除当日频控命中用户。

8.2 完整行为数据留存（第二期 IPW 校正的前提）

完整记录：曝光 + 点击（高意愿自选择群体）和曝光 + 未点击（对照组）。直接用未校正的点击用户特征训练模型，会系统性高估特定剧集类型吸引力，与多样性 OKR 冲突。第二期将用 IPW（逆倾向加权）剥离自选择偏差后，再将净效果特征输入推荐模型。

A/B 期间同步监控 ILD（推荐多样性指标），若低于对照组均值 5% 以上，立即暂停彩蛋特征输入。
8.3 彩蛋后收口的推荐逻辑

彩蛋播完后的续播推荐复用现有大结局后推荐策略，将「解锁了哪类彩蛋（花絮/NG/品牌）」作为置信度增强信号输入：解锁「高甜番外」彩蛋 → 甜宠剧标签置信度 +1，不新建标签，不直接作为模型独立特征。
